FASE 1: Cimientos del Sistema (Usuario, Empresa y Ubicación)
Esta fase establece la arquitectura de la organización y la base geográfica necesaria para las tarifas y el análisis climático.

Órdenes Específicas para la IA
Pautas Clave a Tener en Cuenta

Modelo Freemium: El sistema debe restringir a los usuarios gratuitos a una única entidad de tipo "hogar", mientras que los usuarios Premium pueden gestionar múltiples entidades ("casa", "hogar", "comercio").
1.2. Gestión de Entidades (Propiedades) COMPLETADO
Implementar el CRUD (Crear, Leer, Actualizar, Borrar) para la tabla entities. Asegurar que la creación de entidades requiera la asignación de una ubicación geográfica precisa (locality_id y province_id).
Estructura de la Propiedad: Desarrollar la funcionalidad de Configuración de Estructura (definición de habitaciones o áreas) y Información de Ocupación (cantidad de personas u horarios de funcionamiento) para permitir una representación fiel del espacio a auditar.
1.3. Gestión de Datos Geográficos COMPLETADO
Asegurar la población y el acceso a la tabla de provinces para que el registro de entidades sea coherente y prepare la futura integración de datos climáticos.
La selección de provincia y localidad sienta las bases para una gestión energética personalizada.

Esta fase se centra en la creación detallada del inventario de dispositivos, un punto crítico para la precisión del "Gemelo Digital".

Órdenes Específicas para la IA

El uso de softDeletes es esencial, ya que el motor de cálculo debe considerar equipos eliminados para los análisis de períodos pasados.
FASE 3: Conexión con el Servicio (Contratos y Facturas)
Se configura el contexto tarifario y se inicia el flujo de datos crudos.
Flujo de Trabajo
Órdenes Específicas para la IA
Pautas Clave a Tener en Cuenta

revisar la implementación del CRUD para contracts y supplies. Incluir la tabla utility_companies para registrar la empresa proveedora de energía.
Guía Activa del Usuario: Si una entidad no tiene contrato o suministro, los enlaces para "añadir factura" deben transformarse inteligentemente para guiar al usuario a crear el suministro o contrato primero, evitando callejones sin salida.

revisar la implementación de el registro de facturas (invoices). La carga debe capturar el consumo total (kWh) y el costo total ($) irrefutables.
Desafío de la Desincronización: El sistema debe permitir al usuario ajustar las fechas o datos para lograr la coincidencia entre la carga auditada (inventario) y el consumo facturado real, lo cual es fundamental para la precisión.
3.3. Creación de Snapshots (La "Foto" de Uso) COMPLETADO
Tras cargar una factura, el usuario debe ser redirigido a una pantalla para confirmar los patrones de uso de sus equipos para ese período específico. Esto alimenta la tabla equipment_usage_snapshots.
Propósito: El snapshot es la "foto" de uso (avg_daily_use_minutes, power_watts, has_standby_mode) que se vinculará a la invoice_id y es la base para la calibración y el cálculo del perfil de consumo.

Esta es la fase central donde la plataforma se convierte en un "entrenador personal de eficiencia energética".

Órdenes Específicas para la IA

Este porcentaje es la clave para la calibración continua y debe ser visible para rápida identificación de desviaciones.
